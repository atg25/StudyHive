## **1\. Web Presence**

**Type of Presence:**  
 Primary Website Platform

**Platform Selection:**  
 NJIT Web Server or local hosting environment using HTML, CSS, JavaScript, PHP, and MySQL. GitHub Pages may be used for public-facing documentation if needed.

**Type of Activity:**  
 The website will act as the core operational hub for the platform’s features. Users will be able to register, log in, upload class notes, and view AI-generated summaries or study materials. Subscription tiers may be represented visually as part of the conceptual model but will not include real e-commerce transactions. The activity is focused on demonstrating usability, functionality, and user flow within an academic setting. The site will be accessed during class presentations and peer evaluations to assess system performance and ease of navigation.

## **2\. Academic Social and Collaborative Presence**

**Type of Presence:**  
 Project Communication and Stakeholder Interaction

**Platform Selection:**  
 Canvas or Moodle for academic communication; Microsoft Teams or Google Workspace for internal collaboration.

**Type of Activity:**  
 These platforms will be used to maintain communication with the course instructor and peers. Activities include progress reporting, documentation sharing, meeting scheduling, and gathering user feedback. The communication presence supports iterative improvement by allowing the project team to test features with classmates before final delivery. This presence focuses on structured development in a controlled learning environment. The evaluation from peers and the instructor will help refine the user interface and user experience.

## **3\. Promotional and Demonstration Presence**

**Type of Presence:**  
 Showcase and Presentation

**Platform Selection:**  
 PowerPoint or Google Slides for final presentation; unlisted YouTube or Loom for demo video recording.

**Type of Activity:**  
 This presence is used to present the project’s purpose, design, and capabilities in a clear and professional manner. The promotional materials will help communicate the platform’s value, identify the intended user base, and explain long-term scalability if developed beyond the class. Demonstration videos allow instructors to assess the prototype’s functionality at any time, even outside live demonstration sessions. The presence serves as an academic marketing strategy to emphasize creativity and innovation. All materials will align with the required presentation standards for the course.

## **4\. Testing and Development Presence**

**Type of Presence:**  
 Version Control and Technical Documentation

**Platform Selection:**  
 GitHub for repository management and issue tracking.

**Type of Activity:**  
 GitHub will store and track all versions of source code, supporting a clean and organized development process. It enables collaboration among team members, each being able to contribute code changes while maintaining version control. This presence captures the evolution of the system from early drafts to the final deployed version. The repository will include documentation such as ER diagrams, system architecture, and technical instructions to ensure easy evaluation. At the completion of the course, GitHub will provide a portfolio asset demonstrating technical proficiency
